---
title: "List and manage editions"
description: "Spring Data REST CRUD endpoints for the Edition resource, including listing, creation, update, and deletion."
---

Editions are the top-level resource representing a single First Lego League competition season. All five standard CRUD operations are auto-generated by Spring Data REST.

<Note>
  Write operations (POST, PUT, DELETE) require authentication.
</Note>

---

## List all editions

**GET** `/editions`

Returns a paginated HAL collection of all editions.

### Query parameters

<ParamField query="page" type="number" default="0">
  Zero-based page number.
</ParamField>

<ParamField query="size" type="number" default="20">
  Number of editions per page.
</ParamField>

<ParamField query="sort" type="string">
  Sort expression, e.g. `year,desc`.
</ParamField>

### Example

```bash
curl http://localhost:8080/editions?page=0&size=20
```

```json
{
  "_embedded": {
    "editions": [
      {
        "year": 2025,
        "venueName": "Barcelona",
        "description": "FLL Season 2025",
        "state": "DRAFT",
        "_links": {
          "self": { "href": "http://localhost:8080/editions/1" },
          "edition": { "href": "http://localhost:8080/editions/1" }
        }
      }
    ]
  },
  "_links": {
    "self": { "href": "http://localhost:8080/editions" }
  },
  "page": {
    "size": 20,
    "totalElements": 1,
    "totalPages": 1,
    "number": 0
  }
}
```

## Get an edition

**GET** `/editions/{id}`

Returns a single edition by its numeric ID.

### Path parameters

<ParamField path="id" type="number" required>
  Numeric ID of the edition.
</ParamField>

### Example

```bash
curl http://localhost:8080/editions/1
```

```json
{
  "year": 2025,
  "venueName": "Barcelona",
  "description": "FLL Season 2025",
  "state": "DRAFT",
  "_links": {
    "self": { "href": "http://localhost:8080/editions/1" },
    "edition": { "href": "http://localhost:8080/editions/1" }
  }
}
```

### Response fields

<ResponseField name="year" type="number" required>
  The competition year of this edition.
</ResponseField>

<ResponseField name="venueName" type="string" required>
  Name of the venue where the edition takes place.
</ResponseField>

<ResponseField name="description" type="string" required>
  Human-readable description of the edition.
</ResponseField>

<ResponseField name="state" type="string" required>
  Current lifecycle state. One of `DRAFT`, `OPEN`, or `CLOSED`. Read-only — use `PATCH /editions/{id}/state` to change it.
</ResponseField>

<ResponseField name="_links" type="object" required>
  <Expandable title="properties">
    <ResponseField name="self.href" type="string">
      Canonical URI for this edition.
    </ResponseField>
    <ResponseField name="edition.href" type="string">
      Alias link for this edition.
    </ResponseField>
  </Expandable>
</ResponseField>

## Create an edition

**POST** `/editions` — authentication required

Creates a new edition. The `state` field is always set to `DRAFT` on creation and cannot be supplied in the request body.

### Request body

<ParamField body="year" type="number" required>
  The competition year, e.g. `2025`.
</ParamField>

<ParamField body="venueName" type="string" required>
  Name of the venue. Must not be blank.
</ParamField>

<ParamField body="description" type="string" required>
  Description of the edition. Must not be blank.
</ParamField>

### Example

```bash
curl -X POST http://localhost:8080/editions \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"year": 2025, "venueName": "Barcelona", "description": "FLL Season 2025"}'
```

```json
{
  "year": 2025,
  "venueName": "Barcelona",
  "description": "FLL Season 2025",
  "state": "DRAFT",
  "_links": {
    "self": { "href": "http://localhost:8080/editions/1" },
    "edition": { "href": "http://localhost:8080/editions/1" }
  }
}
```

Returns `201 Created` with the created edition in the response body.

## Update an edition

**PUT** `/editions/{id}` — authentication required

Fully replaces an existing edition's data. The `state` field is read-only and will be ignored if included.

### Path parameters

<ParamField path="id" type="number" required>
  Numeric ID of the edition to update.
</ParamField>

### Request body

<ParamField body="year" type="number" required>
  The competition year.
</ParamField>

<ParamField body="venueName" type="string" required>
  Name of the venue. Must not be blank.
</ParamField>

<ParamField body="description" type="string" required>
  Description of the edition. Must not be blank.
</ParamField>

### Example

```bash
curl -X PUT http://localhost:8080/editions/1 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"year": 2025, "venueName": "Tarragona", "description": "FLL Season 2025"}'
```

Returns `200 OK` with the updated edition.

---

## Delete an edition

**DELETE** `/editions/{id}` — authentication required

Permanently deletes an edition.

### Path parameters

<ParamField path="id" type="number" required>
  Numeric ID of the edition to delete.
</ParamField>

### Example

```bash
curl -X DELETE http://localhost:8080/editions/1 \
  -H "Authorization: Bearer <token>"
```

Returns `200 OK` with an empty body on success.

## Search endpoints

The Edition repository also exposes two search endpoints:

```bash
# Search by year
curl "http://localhost:8080/editions/search/findByYear?year=2025"

# Search by venue name
curl "http://localhost:8080/editions/search/findByVenueName?venueName=Barcelona"
```

Both return a JSON array of matching editions.
