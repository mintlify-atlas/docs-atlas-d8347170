## Prerequisites

Before you begin, make sure you have the following installed:

- **Java 21+** — [Download from adoptium.net](https://adoptium.net/)
- **Maven 3.9+** — [Download from maven.apache.org](https://maven.apache.org/)
- **Git**

<Note>
  No external database is required for local development. The app uses an H2 in-memory database by default. Data is reset each time the app restarts.
</Note>

## Get up and running

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/UdL-EPS-SoftArch-Igualada/first-lego-league-backend.git
    cd first-lego-league-backend
    ```
  </Step>

  <Step title="Build the project">
    Run the Maven build. This compiles the code, runs Checkstyle linting, and executes the full test suite including Cucumber BDD tests.

    ```bash
    mvn package
    ```

    To skip tests during development:
    ```bash
    mvn package -DskipTests
    ```
  </Step>

  <Step title="Start the API server">
    <CodeGroup>
      ```bash Maven CLI
      mvn spring-boot:run
      ```

      ```bash Java JAR
      java -jar target/spring-template-0.0.1-SNAPSHOT.jar
      ```
    </CodeGroup>

    <Tip>
      If you're using VSCode, press **F5** from any file to start debugging directly. The IDE will execute `MainApplication::main`.
    </Tip>

    The server starts on port **8080** by default.
  </Step>

  <Step title="Verify the API is running">
    Make a request to list editions — this endpoint is public and requires no authentication:

    ```bash
    curl http://localhost:8080/editions
    ```

    You should get a HAL response like:

    ```json
    {
      "_embedded": {
        "editions": []
      },
      "_links": {
        "self": { "href": "http://localhost:8080/editions" },
        "profile": { "href": "http://localhost:8080/profile/editions" },
        "search": { "href": "http://localhost:8080/editions/search" }
      },
      "page": {
        "size": 20,
        "totalElements": 0,
        "totalPages": 0,
        "number": 0
      }
    }
    ```
  </Step>

  <Step title="Explore the API interactively">
    Open **Swagger UI** in your browser to browse and test all endpoints:

    ```
    http://localhost:8080/swagger-ui.html
    ```

    Or use the **HAL Explorer** for navigating the hypermedia API:

    ```
    http://localhost:8080/explorer
    ```
  </Step>

  <Step title="Run the tests">
    The test suite uses Cucumber BDD with Gherkin feature files. Run all tests with:

    ```bash
    mvn test
    ```

    Test results include Cucumber reports. Check `target/surefire-reports/` for details.
  </Step>
</Steps>

## Create your first resource

Once the server is running, register a user and create an edition:

### 1. Register a user

```bash
curl -X POST http://localhost:8080/users \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "email": "admin@example.com", "password": "password"}'
```

### 2. Create an edition

```bash
curl -X POST http://localhost:8080/editions \
  -u admin:password \
  -H "Content-Type: application/json" \
  -d '{"year": 2026, "venueName": "Igualada Civic Center", "description": "FLL Igualada 2026 Championship"}'
```

### 3. View the leaderboard

```bash
curl http://localhost:8080/leaderboards/editions/1
```
